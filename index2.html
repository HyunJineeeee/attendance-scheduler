🚀 초기 설치 가이드 (관리자용)

시스템을 처음 구축할 때 한 번만 수행하면 됩니다.

설치 전 사전 준비사항

  • 구글 계정: 스프레드시트 및 캘린더 생성용
  • 구글 캘린더: 출결이라는 이름의 새 캘린더 생성 필수
  • API Key: Groq(또는 OpenAI) API 키 발급

1 슬랙 워크스페이스 및 채널 준비

🤔 슬랙(Slack)이란? 업무용 메신저입니다.
- 워크스페이스: 우리 회사 전체가 모이는 '디지털 사무실 건물'
- 채널(#): 훈련 과정별로 나뉜 '디지털 회의실' (예: #공지-한기대반)
  1. 슬랙(Slack)에 접속하여 워크스페이스를 생성합니다.
  2. 훈련 과정별로 알림을 받을 채널(#)을 미리 만들어둡니다. (예: #공지-한기대, #관리자용)

2 슬랙 앱(Bot) 생성 및 설정

알림을 보낼 '로봇'을 만들고 권한을 부여하는 과정입니다.

  1. Slack API 접속 (로그인 필수) → [Create New App] → [From scratch].
  2. 앱 이름 입력 → 워크스페이스 선택 → [Create App] 클릭.
  3. [필수] 권한 부여: 왼쪽 메뉴 [OAuth & Permissions] → 아래쪽 [Scopes] 섹션으로 이동 → [Bot Token Scopes] 밑의 [Add an OAuth Scope] 버튼 클릭 → incoming-webhook 검색 후 선택.
    (※ 이 과정을 거쳐야 App Home 설정이 가능합니다.)
  4. 봇 이름 설정: 왼쪽 메뉴 [App Home] → 'Your App’s Presence in Slack' 섹션의 [Edit] 클릭 → Display Name(예: OJT알리미) 입력 후 [Save].
  5. 왼쪽 메뉴 [Incoming Webhooks] → 스위치 [On] → 하단 [Add New Webhook to Workspace] 클릭.
  6. 알림 받을 채널을 선택하고 [Allow] 클릭 → 생성된 Webhook URL을 복사해둡니다. (채널별로 반복)

3 훈련 일정표 양식 복사

양식 복사하기 (클릭)

위 버튼을 눌러 시트를 내 구글 드라이브로 복사합니다. (자동화 코드가 포함되어 있습니다)

4 설정 변경 및 배포 (마무리)

  1. 복사된 시트 상단 메뉴 [확장 프로그램] → [Apps Script] 실행.
  2. 코드 최상단의 설정 변수(캘린더명, 슬랙 주소, API 키)를 내 정보로 수정하고 [저장].
  3. [배포] → [새 배포] → [웹 앱] → 권한: [모든 사용자] → 배포 → 웹 앱 URL 복사.
  4. 왼쪽 메뉴 [트리거(시계)] → checkRealTimeAlerts 함수를 1분 간격으로 설정.
  5. 마지막으로 Slack API 사이트의 [Slash Commands] 메뉴에서 /일지, /안전, /질문, /도움말을 생성하고 복사한 웹 앱 URL을 붙여넣으면 끝!
